Remarks

This event can be used to receive a notification that client data has arrived. The Read(Guid, Byte, Int32, Int32) method can then be used to copy data to an existing buffer. In many cases it will be optimal to use an existing buffer instead of subscribing to the ReceiveClientDataComplete event, however, the data that is available after calling the Read(Guid, Byte, Int32, Int32) method will be the original unprocessed data received by the client, i.e., not optionally decrypted or decompressed data.

Argument1 is the ID of the client from which data is received.
Argument2 is the number of bytes received in the buffer from the client.
